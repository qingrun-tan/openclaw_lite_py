# OpenClaw Lite

企业级 AI 智能体框架 - 轻量级 ReAct 智能体实现

## 项目简介

OpenClaw Lite 是一个基于 Python 开发的企业级 AI 智能体框架，采用观察-计划-行动（ReAct）循环模式，支持多种平台接入、多技能扩展和（ReAct）循环模式，支持多种平台接入、多技能扩展和灵活的记忆存储方案。

## 核心模块说明

### 1. 网关层 (Gateway)

**职责**: 统一不同平台的接入方式，将各类平台消息标准化为内部格式。

| 类名 | 说明 | 状态 |
|------|------|------|
| `BaseGateway` | 抽象基类，定义 `receive()` 和 `send()` 接口 | ✅ 实现 |
| `CliGateway` | 命令行适配器，用于本地测试 | ✅ 实现 |
| `WebGateway` | Web API 适配器 | ✅ 实现 |
| `WebSocketGateway` | WebSocket 适配器（流式输出）| ✅ 实现 |

### 2. 智能体层 (Agent)

**职责**: 实现 ReAct 循环 (观察-计划-行动)，协调 LLM 和技能执行。

| 类名 | 说明 |
|------|------|
| `AgentEngine` | 核心引擎，负责完整的 ReAct 循环逻辑 |
| `LLMClient` | 大模型客户端，支持 OpenAI 兼容接口 |
| `Message` | 消息数据类 |
| `LLMResponse` | 响应封装类 |

### 3. 技能层 (Skills)

**职责**: 提供可扩展的功能模块，支持安全护栏机制。

| 技能 | 描述 | 安全机制 |
|------|------|----------|
| `EchoSkill` | 回显消息 | 无 |
| `FinalAnswerSkill` | 直接返回答案 | 无 |
| `ReadFileSkill` | 读取文件 | 路径穿越检查 + 沙箱限制 |
| `WriteFileSkill` | 写入文件 | 路径穿越检查 + 沙箱限制 |
| `AppendFileSkill` | 追加文件 | 路径穿越检查 + 沙箱限制 |
| `ListDirectorySkill` | 列出目录 | 路径穿越检查 |
| `DeleteFileSkill` | 删除文件 | 路径穿越检查 |
| `RunCommandSkill` | 执行命令 | 命令白名单 + 超时控制 |
| `CalculatorSkill` | 数学计算 | 表达式安全检查 |
| `DateTimeSkill` | 时间日期 | 无 |
| `TextProcessSkill` | 文本处理 | 无 |
| `JsonProcessSkill` | JSON 处理 | 无 |

### 4. 记忆层 (Memory)

**职责**: 存储会话历史，支持多实现方案。

| 实现类 | 存储方式 | 适用场景 |
|--------|----------|----------|
| `FileMemory` | 本地 JSON 文件 | 开发/测试环境 |
| `RedisMemory` | Redis 缓存 | 生产环境 (分布式) |

## ReAct 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                      用户输入                                │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│  1. 观察 (Observe)                                            │
│     - 从 Memory 加载历史对话                                  │
│     - 构建 System Prompt (包含可用技能描述)                   │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│  2. 计划 (Plan) - 调用 LLM                                    │
│     - 分析用户意图                                            │
│     - 决定是否需要调用技能                                    │
│     - 返回 JSON 格式决策:                                     │
│       {                                                      │
│         "action": "tool_name/final_answer",                   │
│         "thought": "思考过程",                               │
│         "params": {...},                                     │
│         "final_answer": "答案" (仅当 action=final_answer)     │
│       }                                                      │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│  3. 行动 (Act)                                                │
│     - 如果是 final_answer，返回结果                              │
│     - 如果是 tool_name:                                         │
│       * 从 SkillManager 获取对应技能                               │
│       * 执行技能 (带安全检查)                                   │
│       * 将观察结果反馈给模型，继续循环                            │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│  4. 记忆 (Memory)                                            │
│     - 保存用户输入和 AI 决策到会话历史                         │
│     - 控制历史长度 (避免 Token 溢出)                          │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
               判断是否需要继续循环 → 是 → 回到步骤 2
                         │
                         ▼ 否
                    返回最终结果
```

## 安全机制

### 1. 路径穿越防护
```python
# 防止用户传入 ../../../etc/passwd
abs_path = os.path.abspath(path)
if not abs_path.startswith(sandbox_dir):
    return "安全错误：仅允许访问沙箱目录内的文件"
```

### 2. 命令白名单
```python
ALLOWED_COMMANDS = {"ls", "dir", "echo", "date", "ping", "curl", "cat", "grep"}
```

### 3. 表达式安全检查
```python
# 阻止危险的关键字
DANGEROUS_KEYWORDS = ["import", "exec", "eval", "open", "__", "os", "sys"]
```

### 4. 超时控制
```python
subprocess.run(cmd, timeout=10)  # 10秒超时
```

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.template` 为 `.env` 并填入 API 密钥：

```bash
cp .env.template .env
# 编辑 .env 文件，填入 OPENCLAW_API_KEY
```

### 3. 运行 CLI 模式

```bash
python src/main.py
```

### 4. 运行 Web API 模式

```bash
uvicorn src.api.app:create_app --factory --host 0.0.0.0 --port 18789
```

或使用 Makefile：

```bash
make web
```

### 5. 使用 Docker

```bash
# 构建镜像
make docker-build

# 启动服务
make docker-up

# 停止服务
make docker-down
```

## API 接口

###  POST /chat
核心聊天接口

```bash
curl -X POST http://localhost:18789/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "帮我读取 data/sandbox/test.txt 文件",

    "user_id": "user_001"
```

```bash
curl -X POST http://localhost:18789/sessions
```

## 设计模式

| 模式 | 应用位置 | 说明 |
|------|----------|------|
| 抽象工厂 | `BaseGateway`, `BaseMemory`, `BaseSkill`` | 定义统一接口，支持多种实现 |
| 单例模式 | `Config`, `SkillManager`, `Logger` | 全局唯一实例 |
| 装饰器模式 | `@skill` | 自动注册技能 |
| 策略模式 | Memory 层实现 | 运行时切换存储策略 |
| 依赖注入 | `main.py` | 组装各层组件 |
